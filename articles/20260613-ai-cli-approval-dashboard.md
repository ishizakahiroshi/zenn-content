直近の開発は、9割くらいをAIエージェントに任せています。Claude Code と Codex CLI を、別々のターミナルで何枚も並べて走らせる。そういう回し方が普通になってきました。

便利なんです。便利なんですが、ひとつだけ、地味にずっと困っていたことがありました。承認待ちです。

AIにコマンドの実行やファイルの書き換えを任せると、要所で「これ実行していい？」と聞いてくる。あれ自体はありがたい。勝手に走られたら困る場面も多いので。問題は、それが「いつ」「どの端末で」来るか分からないことでした。

## タブを行ったり来たりして、結局ずっと見張っている

端末を4枚も5枚も開いていると、どれかが手を止めて返事を待っている。でも画面の手前にあるのは別のタブで、止まっているやつは奥に隠れている。

気づかずに5分放置していた、なんてことがざらにありました。AIに任せて自分は別の作業を、と思っても、結局チラチラとタブを巡回している。任せたはずなのに、見張り番をしている。これはおかしい、と思いました。

ターミナル通知やベルも試したんですが、どの端末の話なのかが分からない。鳴っても、結局全部のタブを見に行く羽目になる。

## 「全部の承認待ちを、1画面に集めたい」だけだった

やりたいことは、突き詰めるとそれだけでした。

- どのセッションが今返事待ちなのか、一目で分かること
- その場で承認・却下まで済ませられること
- できれば、手元を離れていてもスマホに通知が飛ぶこと

既存のツールを探したんですが、自分のこの「並列で雑に回す」やり方にぴたっと合うものが見つからなくて、結局自分で作ることにしました。many-ai-cli という名前にしています。

![](./images/2026-06-13_many-ai-cli-v030_fig1.png)

仕組みはそんなに大それたものではありません。各AI CLIを薄くラップして、出力を1つのローカルなHub（管制塔みたいなもの）に集める。Hubはブラウザでダッシュボードを開き、どのセッションが承認待ちかを並べて見せる。承認のボタンを押すと、対応する端末に返事が戻る。CLIそのものの挙動は何も変えていません。通知と承認のGUIを、上にちょこっと乗せているだけです。

## ローカルから出さない、を最初の設計に決めた

ひとつだけ、最初から固く決めていたことがあります。Hubは外に公開しない、ということです。

サーバーは `127.0.0.1`、つまり自分のマシンの中だけに口を開ける。LANにも出さない。リバースプロキシも挟まない。AIにコマンドを実行させる入口を、うっかり外から触れる場所に置くのは、考えるだけで怖かったので。

その代わり、外から使いたいときはSSHのトンネルを掘って、自分の端末の中だけに引き込む。遠回りに見えるけど、入口を増やさないという一点だけは譲りたくなかった。ここは、たぶん正解だったと思っています。

![](./images/2026-06-13_many-ai-cli-v030_fig2.png)

## v0.3.0で、見張りをやめる範囲を少し広げた

そうやって使っているうちに、ぽつぽつ足したものが溜まってきて、今回 v0.3.0 として区切りをつけました。

- スマホから承認できるようにした。PWAとして入れて、承認が来たら通知が飛ぶ。トンネルが切れていても、ntfy経由でお知らせだけは届く
- 接続用のランチャーを、Windowsだけでなく Linux と macOS でも動くようにした。SSHでVPSの中のHubに繋いで、ブラウザを開くまでを1発で
- 画面の一番下に、今動いているセッションのトークン量やコスト、コンテキストの残りを常時出す細い帯を付けた。これは完全に自分が欲しかったやつ
- 声で入力できるようにした。iPhoneのSafariから喋って、PC側のWhisperに投げて返す、みたいな遠回りもできる
- 配布を `pnpm add -g many-ai-cli` で済むようにした。ブラウザでexeを落とす導線をなくして、Windowsの警告に引っかかりにくくする狙いもあります

ぜんぶ「見張りをやめたい」から地続きで生えてきた機能です。最初に大きな設計図があったわけじゃない。困るたびに一個ずつ足した、という方が正直です。

## 小さく、見張らない時間を増やしていく

完成、という感じはまだしていません。承認の検出は端末の見た目をスキャンしている都合で、たまに取りこぼすし、直したそばから別のCLIの新しい画面に気づく、みたいないたちごっこも続いています。これでいいのかは、正直まだよく分かっていません。

それでも、タブを巡回する回数は確実に減りました。任せたものを任せたままにできる時間が、少しずつ増えている。

派手な自動化より、こういう「見張らなくて済む」を一個ずつ増やしていくほうが、自分には合っている気がします。小さく。これからも、そうやって地味に削っていきます。

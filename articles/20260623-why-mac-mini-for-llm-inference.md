## Q1. 推論＝GPU のはずなのに、なんでみんな Mac mini でやってるの？

最初の引っかかりはここでした。Apple Silicon は性能がいい、いいとは聞きます。でもベンチマークの数字を素直に見ると、NVIDIA のハイエンドには勝てていません。なのに「推論なら Mac」と言われる。これがピンと来なかった。

調べていくと、核心は **ユニファイドメモリ（Unified Memory）** にあるという答えに行き着きました。

### 普通の PC の構造

普通の PC では、CPU のメインメモリ（DDR）と GPU の VRAM は物理的に別物です。LLM 推論で一番効くのは「**でかいモデルの重みを VRAM に全部載せられるか**」で、ここがボトルネックになります。

RTX 4060 の 8GB VRAM で 70B が動かない、というのはまさにこれが原因。NVIDIA のコンシューマ GPU は VRAM をケチる傾向があって、70B クラスは載らないんですよね。

### Apple Silicon の構造

Apple Silicon は CPU と GPU が同じメモリを共有します。M2 Ultra なら 192GB、M3 Max でも 128GB 構成が可能で、その全部を GPU が「VRAM として」使えます。

つまり「VRAM 128GB」相当が、NVIDIA で揃えようとすると業務用の数百万円コースになるところ、Mac なら一台で済んでしまう。これが「Mac mini / Mac Studio で動かしてます」勢の正体でした。

### 「性能がいい」の正体

ここでまだしっくり来なかったのが、「Mac は性能がいい」と言われるけど TFLOPS（演算性能）はハイエンド NVIDIA に負けてるよね？という疑問。

これも答えがあって、Mac が優れているのは演算の速さではなくて、別の 3 つです。

1. **メモリ帯域**: M2 Ultra で 800GB/s 級。LLM 推論は演算律速じゃなく **メモリ帯域律速** な場面が多いんですよ。重みを読み出し続ける作業だから
2. **大容量を一台に載せられる**: 載らなければ速さ以前に動かない
3. **電力効率**: アイドル時から推論時まで、NVIDIA 構成より圧倒的に低い

:::message
LLM 推論は計算の速さより「メモリから重みを読み出す速さ」で決まる場面が多い。これが直感に反する逆転を生みます。
:::

つまり「推論＝GPU」という理解は正しい。Mac が選ばれるのは **GPU として速いから** ではなくて、**大きいモデルを現実的な値段・電力で載せられるお手軽手段だから**。

速度で殴るなら NVIDIA、容量で殴るなら Mac、という棲み分けでした。

RTX 4060(8GB) はまさに「速いけど載らない」の典型。逆に 8GB で収まる量子化モデル（7B〜14B の Q4 など）なら 4060 のほうが速い。ここも納得です。

## Q2. 単純に比較ができなくて訳分かんないんだけど、どういうこと？

ここまで来て次にぶつかったのが、「で、結局どっちが上なの？」が出てこないこと。スペック表を見比べても、製品ごとにどの数字が強いかバラバラで、結論が出ない。

「単純に比較できない」のが正解、というのが調べた結論でした。混乱しているのではなく本質を掴んでいる側で、ここは安心していい。

一言で言うと、**LLM 推論の速さ・可否を決める指標が 1 個じゃなくて 3 個あって、製品ごとにどれが強いかバラバラだから**。

### 3 つの軸

| 軸 | 何を決めるか | 例えるなら |
|---|---|---|
| 演算性能 (TFLOPS) | 1 秒あたりの計算量 | エンジンの馬力 |
| メモリ帯域 (GB/s) | 重みを読み出す速さ | 燃料を送るホースの太さ |
| メモリ容量 (GB) | そもそも載るか | ガソリンタンクの大きさ |

PC の世界は「GHz」「コア数」など **1 個の数字で殴り合える** 文化なんですが、LLM 推論は 3 軸が絡むから単純比較が崩壊する。

![](/images/2026-06-23_why-mac-mini-for-llm-inference_fig1.png)

上の図のとおり、LLM 推論の速さ・可否は「演算性能（馬力）」「メモリ帯域（ホースの太さ）」「メモリ容量（タンクの大きさ）」の 3 軸で決まります。製品ごとにどの軸が強いかバラバラなので、1 つの数字で比較しようとすると崩壊します。効きの順番は、タンクに載るか → ホースで流せるか → 最後に馬力、です。

### 具体的に何が起きているか

- **NVIDIA RTX 4060**: 馬力（演算）は強い。でもタンク（容量）が 8GB で小さい。でかいモデルは載らないから動かない
- **Mac (Apple Silicon)**: 馬力はそこそこ。でもタンクが 128GB とデカいし、ホース（帯域）も太い。でかいモデルが載るし、そこそこ速い

ここで効くのが、LLM 推論は **馬力よりホースの太さで速度が決まる** こと。計算してる時間より、巨大な重みをメモリから読み出してる時間のほうが長いから。

だから「演算性能では NVIDIA が勝ってるのに Mac のほうが実用的」という、直感に反する逆転が起きるわけです。

### 結論の出し方

「どっちが上か」を 1 つの数字で答えられないのが正常な状態。質問を **「何 B のモデルを動かしたいか」に変える** とシンプルになりました。

- 7B〜14B の小さいモデル → タンク 8GB で足りる → **4060 が速くて勝ち**
- 70B などデカいの → タンクに載らない → **そもそも 4060 は土俵に上がれない、Mac 一択**

比較するのではなく「**自分の積みたいモデルが載る箱はどれか**」で選ぶ。これが頭の整理の落とし所でした。

## Q3. もうちょっとハイブリッドにした仕組みとか、今後出てきそうじゃない？

3 軸でバラけてるなら、その間を埋める何かが出てきそうなものですよね。Mac の容量と NVIDIA の演算を合わせ持ったみたいな。

答えは「もうガッツリ出てきてる」でした。業界は今まさにその方向に動いていて、知らなかったのは自分だけだったというオチ。

### 分断は、もう埋めにいかれている

「NVIDIA = 演算・帯域強いけど容量足りない / Mac = 容量あるけど演算弱い」という構図に対して、**AMD が「PC なのにユニファイドメモリ」をぶつけてきた**のが今の最大の動きです。それが **Strix Halo（Ryzen AI Max+ 395）**。

中身はまさにハイブリッドで、

- 1 つのチップに CPU・GPU(iGPU)・NPU を全部載せ
- 128GB のユニファイド LPDDR5X メモリを積み、うち最大 96GB を GPU が VRAM として使える
- つまり **Mac のユニファイドメモリ方式を、x86 PC の世界に持ち込んだ**

ということになっています。Mac 的な容量の強みと、PC 的なエコシステム（Windows / Linux / Ollama / LM Studio）を両取りしにきている。

### 「3 軸が製品ごとにバラバラ」がさらに進む

ただ、Strix Halo も演算が万能になったわけじゃないんです。3 軸で見ると次のとおり。

|  | 容量 | 帯域 | 演算 |
|---|---|---|---|
| RTX 4060 (8GB) | 小さい | 速い | 強い |
| Mac Studio (128GB) | デカい | 太い (800GB/s 級) | そこそこ |
| Strix Halo (128GB) | デカい | 細い (256GB/s 級) | そこそこ |

![](/images/2026-06-23_why-mac-mini-for-llm-inference_fig2.png)

並べて見ると、RTX 4060 は演算と帯域は強いが容量で土俵に上がれず、Mac Studio は 128GB の容量と 800GB/s 級の太いホースで大モデルを流せ、Strix Halo は同じ 128GB でも帯域は細いが価格で粘る、という棲み分けになっています。同じ「128GB」でも、Mac は帯域で殴り、Strix Halo は価格で殴っているのが図から読み取れます。

Strix Halo のユニファイドメモリ帯域は約 256GB/s で、ディスクリート VRAM の 1,000GB/s 超よりずっと低い。だから小さいモデルでの 1 トークンあたりの速度は劣ります。

でもデカいモデルはディスクリート GPU だとそもそも載らない。Mac より帯域は細いけど、値段で殴ってきていて、128GB 構成で 1,499〜2,500 ドルと、同等の Mac Studio M4 Max を 1,500〜2,000 ドル下回るらしい。

### 今後さらに来そうな具体的動き

1. **AMD の次世代（2026 Q3）**: Ryzen AI Max PRO 400 シリーズで、ユニファイドメモリを最大 192GB、VRAM 割り当て 160GB まで引き上げ
2. **クラスタ化でデカさを稼ぐ**: Ryzen AI Max+ を 4 ノードつないで、1 兆パラメータ級モデル (Kimi K2.5) をローカル推論するデモを AMD 自身が公開している。「容量が足りないなら箱を増やせばいい」という流れ
3. **MoE（Mixture of Experts）との相性**: トークンごとに一部パラメータしか使わない MoE モデルとの相性が良くて、Qwen3-30B-A3B で毎秒 52 トークン。「デカいけど全部は計算しない」構造が主流になれば、容量重視ハードの価値がさらに上がる

:::message
全軸最強の 1 台が出るのではなく、容量の壁をユニファイドメモリ＋クラスタ＋MoE で回避する流派が育っている、という形のようです。
:::

## 自分用の整理

頭の中はこんな感じで落ち着きました。

- 「推論＝GPU」は正しい。ただし GPU を「演算の速さ」だけで選ぶと外す
- LLM 推論は **演算より、メモリ帯域・容量で勝負が決まる場面が多い**
- 1 つの数字で比較しようとしない。「**何 B を動かしたいか**」を先に決めると選択肢が絞れる
- Mac は「容量重視」流派の代表、Strix Halo はその x86 版、NVIDIA は「演算重視」流派の代表
- 今後は容量側の選択肢がさらに広がる方向（次世代 Strix Halo、クラスタ、MoE）

自分が次にハード買うときは、まず「動かしたいモデルのサイズ」を決めて、それが載るかどうかを見る。速度はそのあと。順番だけ間違えないようにしたいです。

価格・ベンチマーク値は時期で大きく変動するので、ここに書いた数値は執筆時点（2026 年 6 月）のスナップショットとして見ておいてください。同じ問いで詰まっている人の整理の足がかりになれば、嬉しいです。
